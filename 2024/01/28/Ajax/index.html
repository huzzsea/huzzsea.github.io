第一节 Ajax概述

  • Ajax 简介：全称为 async Javascript and XML，即异步 JavaScript 和 XML，是一种用于创建快速动态网页的技术，它通过在后台与服务器进行少量数据交换，可以使网页实现异步更新，是前后台交互的工具
  • 同步与异步的区别
    • 同步提交：当用户发送请求时，当前页面不可以使用，服务器响应页面到客户端，响应完成，用户才可以使用页面
    • 异步提交：当用户发送请求时，当前页面还可以继续使用，当异步请求的数据响应给页面，页面把数据显示出来
  • Ajax 的特点
    • 不需要插件的⽀持，原⽣ JavaScript 就可以使⽤
    • ⽤户体验好，不需要刷新⻚⾯就可以更新数据
    • 减轻服务端和带宽的负担
  • Ajax 的工作原理：客户端发送请求，请求交给 Ajax，Ajax 把请求提交给服务，服务器进行业务处理，把响应数据交给 Ajax 对象，Ajax 对象接收数据，由 JavaScrip 把数据写到页面上

第二节 Ajax基本步骤

  • 创建 XMLHttpRequest 对象

  • 创建 HTTP 请求

    • method：该参数用于指定 HTTP 的请求方法，一共有 get、post、head、put、delete 五种方法
    • URL：该参数用于指定 HTTP 请求的 URL 地址，可以是绝对 URL，也可以是相对 URL
    • flag：该参数为可选，参数值为布尔型，用于指定是否使用异步方式，true 表示异步、false 表示同步，默认为 true
    • name：该参数为可选参数，用于输入用户名，如果服务器需要验证，则必须使用该参数
    • password：该参数为可选，用于输入密码，若服务器需要验证，则必须使用该参数

  • 设置响应 HTTP 请求状态变化的函数

    • 设置当对象状态改变时调用的函数

    • 定义响应函数

  • 设置获取数据的语句

  • 发送 HTTP 请求

    • data 参数的格式与在 URL 中传递参数的格式类似
    • 如果请求的数据不需要参数，即可以使用 nul 来替代

  • 局部更新：在通过 Ajax 的异步调用获得服务器端数据之后，可以使用 JavaScript 或 DOM 来将网页中的数据进行局部更新

第三节 Ajax状态码

  • XMLHttpResponse 的状态

    • 未初始化状态：在创建完 XMLHttpRequest 对象时，该对象处于未初始化状态
    • 初始化状态：在创建完 XMLHttpRequest 对象后使用 open() 方法创建了 HTTP 请求时，该对象处于初始化状态
    • 发送数据状态：在初始化 XMLHttpRequest 对象后，使用 send() 方法发送数据时，该对象处于发送数据状态
    • 接收数据状态：Web 服务器接收完数据并进行处理完毕之后，向客户端传送返回的结果
    • 完成状态：XMLHttpRequest 对象接收数据完毕后，此时接收完毕后的数据存入在客户端计算机的内存中，可以使用 responseText 属性或responseXml 属性来获取数据

  • Ajax 状态码

    Ajax状态码 状态
    0 未初始化状态
    1 初始化状态
    2 发送数据状态
    3 接收数据状态
    4 完成状态
