Git

第一节 Git概述

  • Git 简介：一个免费的、开源的分布式版本控制系统，可以快速高效地处理从小型到大型的各种项目，Git 易于学习，体积小，性能极快，具有廉价的本地库，方便的暂存区域和多个工作流分支等特性
  • 版本控制工具
    • 集中式版本控制工具：有一个单一的集中管理的服务器，保存所有文件的修订版本，而协同工作的人们都通过客户端连到这台服务器，取出最新的文件或者提交更新，例如 CVS,SVN,VSS 等
    • 分布式版本控制工具：客户端提取的不是最新版本的文件快照，而是把代码仓库完整地镜像下来，任何一处协同工作用的文件发生故障，事后都可以用其他客户端的本地仓库进行恢复，例如 Git，Mercurial，Bazaar 等
  • Git 工作机制
    • 工作区：存放项目代码的地方
    • 暂存区：用于临时存放代码的改动，它事实上是一个文件，保存即将提交的文件列表信息，一般存放于 index 文件中
    • 仓库区：安全存放数据的位置，包含提交的所有版本数据

第五节 IDEA集成Git

  • Git 忽略文件：与项目的实际功能无关，不参与服务器上部署运行，把它们忽略掉能够屏蔽 IDE 工具之间的差异
  • 分支整合
    • 变基：是将一个分支的更改应用到另一个分支的过程，不同于合并操作，它会将分支上的提交重新应用在目标分支上
    • 合并：是将一个分支的更改合并到另一个分支中，它会创建一个新的提交，包含两个分支的更改，合并会在分支历史中保留各个分支的独立提交历史
  • 获取变更
    • 提取：仅会从远程仓库中获取最新的提交记录，不会自动将其应用到本地分支，可以查看远程仓库上的变更情况，然后再决定是否将其合并到本地分支上
    • 拉取：先从远程仓库中获取最新的提交记录，然后将这些提交记录应用到本地分支上，可以更新本地分支，使其与远程分支保持同步，在执行拉取操作时，还可以选择将远程分支合并到当前本地分支上
